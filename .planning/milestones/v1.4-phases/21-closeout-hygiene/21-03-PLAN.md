<objective>
Eliminate all 32 `no-console` ESLint warnings across 9 source files by applying per-site `eslint-disable-next-line no-console` directives with a category-tagged rationale (D-LOCK-2). Logger abstraction is deferred to v1.5; the per-site disable approach matches the corpus size (32 sites, scattered, 1–8 per file) and creates reviewer-visible attestation per Pattern 9 (Type Names as Governance).

Per D-LOCK-2-AMEND-A, **Step 0 produces a 32-row categorization manifest BEFORE any disable lands**. Manifest format: `file:line | method | category | rationale`. The 4 allowed categories are:
- `user-facing-output` — console writes that ARE the CLI/command's intended output (validation results, routing decisions, briefing emit)
- `degradation-warning` — non-fatal degradation paths where the program continues with reduced function (config-load failure, watcher loss, parse fallback)
- `diagnostic` — always-on observability emit (audit logs, structured event traces) intentionally going to stderr
- `last-resort-error` — error paths where the program is about to fail and is communicating the failure mode before exit/return

Manifest is archived as part of `21-03-SUMMARY.md` and serves as the audit trail. Reviewer validates disables against manifest.

Per Lock 5: only `eslint-disable-next-line` comments added. No logic changes. No console.* calls removed.

Purpose: Closeout hygiene that makes the lint warning surface zero before v1.4 close, while preserving the existing console.* behavior and creating a clean baseline for v1.5 logger abstraction.
Output: 32 disabled sites + 1 manifest archive in 21-03-SUMMARY.md.
</objective>
