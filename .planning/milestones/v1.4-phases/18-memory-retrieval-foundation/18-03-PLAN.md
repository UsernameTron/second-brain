```javascript
}
```

       ```javascript
       /**
       ```

    2. Implement `readMemory()`:
       - `try { fs.promises.readFile(MEMORY_FILE(), 'utf8') } catch (err) { if (err.code === 'ENOENT') return []; throw err; }` — only ENOENT returns empty; other errors propagate.
       - Split on `^### ` boundaries (regex `/\n### /` with careful first-chunk handling).
       - For each chunk, parse the header line: split on ` · `. If fewer than 3 segments, emit `process.stderr.write` warning and skip.
       - Extract date (segment 0 trimmed), category-from-header (segment 1), sourceRef-short-from-header (segment 2) — but the AUTHORITATIVE sourceRef comes from `source-ref::` field body.
       - Split chunk body on `\n`. Collect `::`-style fields into a map. Everything before the first `::` line is `content` (trim).
       - Use `contentHash` from `content_hash::` if present, else `computeHash(content)`.
       - Build `id = contentHash` for stability.
       - Return the entry array (malformed skipped).

    3. Implement `searchMemoryKeyword(query, options = {})`:
       - Call `await readMemory()` to load entries.
       - Preprocess query: extract negation tokens (words prefixed with `-`), extract quoted phrases (content between `"`), leave remaining tokens as the minisearch query string.
       - Build a fresh MiniSearch instance with the config from the interfaces block. `addAll(entries)` once.
       - Run `index.search(cleanQuery)`.
       - Post-filter: drop results whose `content` contains any negation token (case-insensitive substring match); drop results that do NOT contain every quoted phrase.
       - Apply `options.category` exact match, `options.since` (`entry.date >= options.since`), `options.until` (if supplied — optional, not required by MEM-SEARCH-KW-01).
       - For each surviving result, compute a snippet: find the first query-term match in `content`, slice ± ~50 chars around it, cap at 100 chars. If no match position (e.g. phrase-only query), snippet = `content.slice(0, SNIPPET_MAX)`.
       - Return the array ordered by minisearch score desc.

    4. Implement `getMemoryEcho(connectorResults, options = {})`:
       - Guard: if `connectorResults == null`, return `{ entries: [], score: 0 }`.
       - Extract topics: concat `connectorResults.calendar?.data?.map(e => e.summary)` when `calendar.success`, and `connectorResults.gmail?.data?.map(e => e.subject)` when `gmail.success`. Filter falsy, lowercase, strip punctuation via `.replace(/[^\w\s]/g, ' ')`, split into tokens, dedupe.
       - If topic token count is 0, return `{ entries: [], score: 0 }`.
       - Load entries via `await readMemory()`.
       - For each entry: lowercased `haystack = entry.content + ' ' + entry.category + ' ' + (entry.tags || '')`. Count how many topic tokens appear as substrings in haystack. `score = matches / totalTopicTokens`.
       - Select top 5 entries with `score >= (options.threshold ?? DEFAULT_ECHO_THRESHOLD)`.
       - For each selected entry: build snippet (same helper as searchMemoryKeyword — extract to a private `_snippet(content, term)` helper), include `category`, `sourceRef`, `score`.
       - Return `{ entries, score: maxScore }`.

    5. Export: `module.exports = { readMemory, searchMemoryKeyword, getMemoryEcho };`
