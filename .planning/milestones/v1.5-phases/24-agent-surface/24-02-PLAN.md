    ```javascript
    'use strict';
    /**
     * src/today/memory-health.js
     *
     * Anomaly detector for memory health. Reads parsed daily-stats rows and
     * returns a markdown body when any anomaly is detected, or null when
     * everything is normal. The null return suppresses the ## Memory Health
     * section entirely (following the Memory Echo precedent from Phase 18).
     *
     * Requirement: AGENT-MEMORY-01
     * @module today/memory-health
     */

    /**
     * @param {Array<object>} rows — parsed from readDailyStats(), ascending date order
     * @param {object} thresholds — from config.memoryHealth
     * @param {boolean} [thresholds.enabled=true]
     * @param {number} [thresholds.streakDays=3]
     * @returns {string|null} markdown body or null
     */
    function computeMemoryHealth(rows, thresholds) { ... }
    ```

    Implementation logic:
    - If `!thresholds || thresholds.enabled === false`, return `null`
    - `const streak = (thresholds && thresholds.streakDays) || 3;`
    - If `rows.length < streak`, return `null` (sparse data guard)
    - Take the last `streak` rows from the array (tail slice: `rows.slice(-streak)`)
    - Helper: `_num(val)` — returns `Number(val) || 0` (handles undefined, empty string, NaN)
    - Check 4 conditions against the tail rows:

    **Zero-promotion streak:** every row in tail has `_num(row.promotions) === 0`
    **Backlog growth trend:** proposals strictly increases across all consecutive pairs in tail AND no row in tail has `_num(row.promotions) > 0`
    **Recall usage drop:** every row in tail has `_num(row.recall_count) === 0`
    **Vault plateau:** `_num(tail[0].total_entries)` equals `_num(tail[tail.length - 1].total_entries)` AND all intermediate values are the same

    - Collect triggered condition names into an array `alerts`
    - If `alerts.length === 0`, return `null`
    - Build markdown body:
      ```
      **Anomalies detected** (last {streak} days):
      
      - {condition description for each alert}
      
      Review `daily-stats.md` and consider running `/promote-memories` or `/recall` to re-engage the memory pipeline.
      ```

    Condition descriptions:
    - zero-promotion: `Zero memory promotions for {streak} consecutive days`
    - backlog-growth: `Memory proposal backlog growing without promotions`
    - recall-drop: `No /recall usage for {streak} consecutive days`
    - vault-plateau: `Vault entry count unchanged for {streak} consecutive days ({count} entries)`

    Export: `module.exports = { computeMemoryHealth };`

    4. Run tests — they MUST pass (GREEN).
