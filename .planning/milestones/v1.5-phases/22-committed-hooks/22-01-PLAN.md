<objective>
Create the git pre-commit hook infrastructure that enforces two classes of defects at commit time: (1) schema validation for config/*.json files and daily-stats.md frontmatter via AJV, and (2) vault LEFT/RIGHT boundary enforcement for staged files.

Purpose: These checks currently rely on runtime code (config-validator.js, vault-gateway.js) and developer discipline. Moving them to the git layer means defects are caught before they land on master, not after.

Output: A `hooks/` directory (committed to the repo, symlinked or configured via git) with a pre-commit orchestrator and two Node.js validation scripts, plus a new JSON Schema for daily-stats.md frontmatter and tests for both hooks.
</objective>

    - Test 2: A staged file path starting with a LEFT-side segment (e.g., "ABOUT ME/identity.md") is blocked with a descriptive error mentioning the LEFT/RIGHT boundary
    - Test 3: Files outside both LEFT and RIGHT (e.g., "src/foo.js", ".planning/STATE.md") are ignored (pass through — they are repo files, not vault files)
    - Test 4: When VAULT_ROOT is not set or the vault directory is not a git subpath, all files pass (the hook is a no-op when not applicable)
  </behavior>
  <action>
    1. Create `hooks/pre-commit-vault-boundary.js`:
       - Read `config/vault-paths.json` to get LEFT and RIGHT arrays
       - Get staged files via `git diff --cached --name-only --diff-filter=ACM`
       - For each staged file: extract the first path segment. If it matches a LEFT-side entry from vault-paths.json, that is a violation — print the file path and "vault boundary violation: LEFT-side path '{segment}' must not be committed by automation" to stderr, set exit code 1
       - Files whose first segment matches RIGHT or matches neither LEFT nor RIGHT pass through (repo-internal files like src/, test/, config/ are not vault paths)
       - Exit 0 if no violations, exit 1 if any violation found
       - The check is lightweight: string comparison of first path segment against the LEFT array. No filesystem access to the vault needed.

    2. Create `hooks/pre-commit` (shell script, chmod +x):
       ```bash
       #!/usr/bin/env bash
       set -euo pipefail
       # Pre-commit hook orchestrator for Second Brain
       # Runs: schema validation, vault boundary check
       # Install: ln -sf ../../hooks/pre-commit .git/hooks/pre-commit

       HOOK_DIR="$(cd "$(dirname "$0")" && cd ../../hooks && pwd)"

       echo "[pre-commit] Running schema validation..."
       node "$HOOK_DIR/pre-commit-schema-validate.js"

       echo "[pre-commit] Running vault boundary check..."
       node "$HOOK_DIR/pre-commit-vault-boundary.js"

       echo "[pre-commit] All checks passed."
       ```
       - The script references `../../hooks/` relative to `.git/hooks/` where the symlink lives
       - Each sub-hook is a separate node invocation so failures are isolated

       - Use `config/vault-paths.json` LEFT/RIGHT arrays directly
       - Assert that LEFT paths are blocked, RIGHT paths pass, and non-vault paths are ignored
