    **1. Create src/pipeline-infra.js** with these exports:

    **generateCorrelationId()** — Returns UUID v4 string (use `crypto.randomUUID()`). Per D-42, this is the correlation ID for pipeline stages.

    **createHaikuClient(options)** — Wrapper around `@anthropic-ai/sdk`. Returns object with:
    - `classify(systemPrompt, userContent, options)` — Calls Anthropic API with model `claude-haiku-3-5-sonnet` (or configured model), returns `{ success: true, data: parsedJSON }` or `{ success: false, error: string, failureMode: 'api-error'|'timeout'|'parse-error' }`. Parses response as JSON. On JSON parse failure, returns failureMode `parse-error`. On API error/timeout, returns corresponding failureMode per D-36 taxonomy. Never throws — all errors captured in return value.
    - Uses `sanitizeTermForPrompt` from content-policy.js for any interpolated values in prompts.
    - Logs call metadata via `logDecision` from vault-gateway.js.

    **createSonnetClient(options)** — Same interface as createHaikuClient but uses Sonnet model. For D-04 escalation path.

    **writeDeadLetter(inputBody, failureMode, correlationId, metadata = {})** — Writes a dead-letter file to `proposals/unrouted/` via vault-gateway's `vaultWrite`. File format:
    ```yaml
    ---
    created: <ISO-8601 with timezone>
    failure-mode: <one of 7 modes from D-36>
    correlation-id: <UUID>
    status: unrouted
    retry-count: 0
    original-source: <metadata.source or 'unknown'>
    ---
    <inputBody verbatim>
    ```
    Filename: `unrouted-<YYYYMMDD>-<HHmmss>-<first 8 chars of correlationId>.md`

    **loadPipelineConfig()** — Reads and validates config/pipeline.json against schema. Returns parsed object. Throws on validation failure.

    **loadTemplatesConfig()** — Reads and validates config/templates.json against schema. Returns parsed object. Throws on validation failure.

    - Replace the existing `watchConfig` function that uses `fs.watch` with `chokidar.watch(configDir, { ignoreInitial: true })`
    - On change: validate JSON, if valid do atomic in-memory swap of config, emit `config:reloaded` event via EventEmitter
    - On invalid JSON: log warning, keep previous config (fail-closed)
    - Watch the entire `config/` directory (vault-paths.json, pipeline.json, templates.json, excluded-terms.json)
    - Export the EventEmitter as `configEvents` from vault-gateway.js
