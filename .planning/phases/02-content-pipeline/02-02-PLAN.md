  <action>
    Create src/classifier.js with exports: classifyInput, runStage0, runStage1, runStage2.

    **runStage0(content, correlationId):**
    - Call `checkContent(content, excludedTerms)` from content-policy.js
    - If `action === 'BLOCK'`: return `{ blocked: true, reason }` — caller exits immediately per D-41
    - If checkContent throws (internal failure): return `{ blocked: false, deadLetter: true, failureMode: 'exclusion-unavailable' }` per D-36. Fail-closed: dead-letter rather than skip exclusion gate.
    - If allowed: return `{ blocked: false }`

    **runStage1(content, correlationId, options = {}):**
    - Uses createHaikuClient from pipeline-infra.js
    - System prompt: "You are a voice-authenticity classifier. Determine if this text was written by or sounds like it was written by the user personally (LEFT) or is structured/agent-generated/external data (RIGHT)."
    - Include D-59 operational rules in the system prompt:
      - LEFT signals: first-person voice, personal opinions/reflections, relationship context, emotional content, identity statements, career narratives, communication style preferences
      - RIGHT signals: structured data, agent-generated content, research summaries, extracted/synthesized information, technical reference, briefing output, template-driven content
      - Mixed content: classify by dominant voice; if roughly equal, classify LEFT (false-LEFT recoverable via /reroute)
      - Short inputs (< pipeline.classifier.shortInputChars chars) with no clear voice signal: default RIGHT with reduced confidence
    - Expected JSON response: `{ "side": "LEFT"|"RIGHT", "confidence": 0.0-1.0, "rationale": "..." }`
    - Stage 1 does NOT escalate to Sonnet per D-04
    - On API/parse failure: return `{ side: null, confidence: 0, failureMode: <from Haiku client> }`

    **runStage2(content, stage1Result, correlationId, options = {}):**
    - Reads vault-paths.json to build label list
    - If stage1Result.side === 'LEFT': labels are LEFT directories: `["ABOUT ME", "Daily", "Relationships", "Drafts"]`
    - If stage1Result.side === 'RIGHT': labels are RIGHT directories (excluding proposals — classifier never routes to proposals directly per D-02)
    - System prompt: "Given this text classified as {side}, pick the most appropriate subdirectory." Lists labels with one-line descriptions.
    - Expected JSON: `{ "directory": "string", "confidence": 0.0-1.0, "rationale": "..." }`
    - If Haiku confidence < pipeline.classifier.sonnetEscalationThreshold (0.8): escalate to Sonnet per D-04
    - If Sonnet confidence >= pipeline.classifier.sonnetAcceptThreshold (0.7): accept Sonnet result
    - If Sonnet confidence < 0.7: return `{ needsInteractive: true, topCandidates: [top2], confidence: sonnetConfidence }`
    - On API/parse failure: return `{ directory: null, confidence: 0, failureMode: <from client> }`

    **classifyInput(content, options = { interactive: true }):**
    - Generates correlationId via generateCorrelationId()
    - Runs Stage 0 -> Stage 1 -> Stage 2 sequentially per D-40
    - At each stage, checks for failure and either dead-letters or prompts user depending on options.interactive
    - Non-interactive mode (options.interactive === false): if Stage 1 confidence < threshold, call writeDeadLetter with failureMode 'non-interactive-ambiguous' per D-03
    - Logs full instrumentation per D-06: `{ correlationId, inputLength, stage1: { side, confidence }, stage2: { directory, confidence }, sonnetEscalated: bool, destination: string }`
    - Returns: `{ correlationId, blocked, side, directory, suggestedLeftPath (if LEFT), confidence, needsInteractive, failureMode, deadLettered }`

    When stage1.side === 'LEFT', the `directory` from Stage 2 is stored as `suggestedLeftPath` (advisory only per D-02). The actual write target is always `proposals/left-proposals/`.

    **1. Create src/note-formatter.js** with exports: formatNote, formatLeftProposal, generateFilename, extractTemplateFields.

    **formatNote(inputBody, classificationResult, options = {}):**
    - Builds YAML frontmatter per D-07:
      ```yaml
      ---
      created: <ISO-8601 with timezone offset per D-11>
      source: <options.source or 'cli'>
      domain: <classificationResult.directory>
      routed-by:
        stage-1: { side: <>, confidence: <> }
        stage-2: { directory: <>, confidence: <>, sonnet-escalated: <bool> }
      filename-basis: <'first-line'|'haiku-generated'|'user-provided'>
      tags: []
      ---
      ```
    - Appends raw input body verbatim (never modify user content)
    - If classificationResult.directory matches a key in templates.json domain-templates, call extractTemplateFields() and append overlay fields after frontmatter, before body. Template fields that extraction couldn't fill are left empty (never invent values per D-09).

    **extractTemplateFields(inputBody, domainKey, correlationId):**
    - Lightweight Haiku pass per D-09 to extract known fields for the matched domain
    - Returns object with field values (empty string for low-confidence fields)
    - On API failure, returns all fields empty (enrichment failure never blocks per D-39)

    **formatLeftProposal(inputBody, classificationResult, options = {}):**
    - Builds left-proposal frontmatter per D-13:
      ```yaml
      ---
      created: <ISO-8601>
      type: left-proposal
      source: <options.source or 'cli'>
      suggested-left-path: <classificationResult.suggestedLeftPath>
      proposal-action: <create|append>
      routed-by:
        stage-1: { side: LEFT, confidence: <> }
        stage-2: { directory: <>, confidence: <> }
      status: pending
      ---
      ```
    - If Stage 2 picked "Daily", set suggested-left-path to `Daily/{YYYY-MM-DD}.md` and proposal-action to `append` per D-14. If today's daily note doesn't exist, set proposal-action to `create`.
    - Appends suggested content and review checklist per D-13: `## Review\n- [ ] Accept\n- [ ] Edit\n- [ ] Reject\n- [ ] Re-route`

    **generateFilename(inputBody, options = {}):**
    - If `options.name`: use it (filename-basis: user-provided), sanitize (remove illegal chars, replace spaces with hyphens, max 60 chars) per D-10
    - If input < pipeline.filename.maxLength chars and looks title-like (no newlines, no sentence-ending punctuation): use first line as filename (filename-basis: first-line)
    - Else: Haiku generates 4-8 word filename (filename-basis: haiku-generated)
    - Append `.md` extension
    - Collision check: if file exists at target path, append -2, -3, etc.

    **2. Create src/new-command.js** with export: runNew.

    **runNew(input, options = { interactive: true, name: undefined, source: 'cli' }):**
    - If no input and options.interactive: prompt user for input (readline)
    - If no input and not interactive: return error
    - Generate correlationId
    - **Stage 0:** Call classifier.runStage0(input, correlationId). On BLOCK: print block reason, exit immediately per D-41. On exclusion-unavailable: dead-letter and notify per D-36.
    - **Stage 1:** Call classifier.runStage1(input, correlationId, { interactive: options.interactive }). If confidence < threshold and interactive: prompt user "Is this personal/voice content (LEFT) or reference/data (RIGHT)?" If confidence < threshold and non-interactive: dead-letter with non-interactive-ambiguous.
    - **Stage 2:** Call classifier.runStage2(input, stage1Result, correlationId). Handle Sonnet escalation and interactive pick per D-04.
    - **Format:** If LEFT: formatLeftProposal(). If RIGHT: formatNote(). Generate filename via generateFilename().
    - **Write:** Call vaultWrite(targetPath, formattedContent). Target path: `proposals/left-proposals/<filename>` for LEFT, `<directory>/<filename>` for RIGHT.
    - **Template extraction:** If RIGHT and domain is templated (briefings, job-hunt, interview-prep): extractTemplateFields runs as part of formatNote (Stage 3 per D-40).
    - **Output:** Print `"Routed to <targetPath>"` with correlation ID. If dead-lettered: print `"Dead-lettered to proposals/unrouted/<filename> (failure-mode: <mode>)"`.
    - On any Stage 1-5 failure: dead-letter via writeDeadLetter per D-35. Never lose captures that cleared Stage 0.

    Note: Wikilink generation (Stage 4) is NOT in this plan. It is handled by Plan 03 which integrates into /new post-write.
