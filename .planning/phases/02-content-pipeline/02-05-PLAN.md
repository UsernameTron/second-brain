    - Read proposals/memory-proposals.md
    - If total candidates > pipeline.promotion.proposalArchiveThreshold (100):
      - Move all non-pending candidates (accepted, rejected, duplicate, deferred) to `memory-proposals-archive/YYYY-MM.md` grouped by captured_at month

    - Call readProposals() from memory-proposals.js
    - Filter for candidates where the `accept` checkbox is checked (`- [x] accept`)

    **Step 3 — Batch and dedup:**
    - Sort accepted candidates by confidence descending per D-32
    - Take top N (options.max, default 10)
    - Excess accepted candidates: update status to `deferred` in proposals file
    - For each candidate: compute content_hash, check against memory.md + memory-archive/*.md per D-33. Match → mark as `duplicate-of-existing-memory`, skip.

    **Step 4 — Write to memory.md per D-28/D-29:**
    - Read current memory.md (or create if missing)
    - Find or create month-level section: `## YYYY-MM` (newest month at top per D-28)
    - For each candidate, append entry:
      ```
      ### YYYY-MM-DD . CATEGORY . source-ref-short

      <one-paragraph prose body from candidate content>

      category:: CATEGORY
      source-ref:: <full source ref>
      tags:: <comma-separated tags>
      added:: <ISO-8601 of promotion time>
      related:: <wikilinks if proposed>
      ```
    - Write via vaultWrite to `memory/memory.md`

    **Step 5 — Update proposals file:**
    - Update promoted candidates: status:: accepted (confirmed)
    - Update deferred candidates: status:: deferred
    - Update duplicates: status:: duplicate-of-existing-memory
    - Write updated proposals file

    **Step 6 — Memory archive check (D-34):**
    - Check memory.md size (bytes) and entry count
    - If > pipeline.promotion.archiveSizeThresholdKB * 1024 or > pipeline.promotion.archiveEntriesThreshold:
      - Archive oldest complete year(s) to `memory-archive/YYYY.md`
      - Retain current + previous year minimum
      - Write archive file via vaultWrite

    - Return: { promoted: N, deferred: N, duplicates: N, rejected: N, skipped: N, archived: bool }
