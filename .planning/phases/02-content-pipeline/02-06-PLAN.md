  <action>
    Create src/lifecycle.js with exports: retryDeadLetters, archiveStaleLeftProposals.

    **retryDeadLetters():**
    - Load pipeline config for retry.delayMinutes (15) and retry.maxAttempts (3)
    - List all .md files in proposals/unrouted/ (exclude subdirectories promoted/, rerouted/)
    - For each file:
      - Parse YAML frontmatter to get failure-mode, retry-count, status, created/last-retry timestamp
      - **Skip if:** status is "frozen", or failure-mode is non-retryable (parse-error, confidence-floor, gate-rejection, non-interactive-ambiguous per D-37), or retry-count >= maxAttempts, or last modification < delayMinutes ago
      - **Retry:** Extract original input body (strip frontmatter). Re-invoke the pipeline:
        - For api-error/timeout: re-run classifyInput(body, { interactive: false }) — the full Stage 0-5 pipeline
        - For exclusion-unavailable: re-run runStage0 only first; if Stage 0 now succeeds, continue with classifyInput
      - **On success:** Write formatted note to classified destination via same logic as /new. Move original dead-letter to proposals/unrouted/promoted/ with metadata (promoted-at, promoted-to, promoted-by: auto-retry)
      - **On failure:** Increment retry-count in frontmatter. Add last-retry: ISO-8601 timestamp. If retry-count now equals maxAttempts: set status: frozen per D-37. Write updated frontmatter back to original file.
    - Return: { retried, succeeded, failed, frozen, skipped }

    **archiveStaleLeftProposals():**
    - Load pipeline config for leftProposal.autoArchiveDays (14) per D-16
    - List all .md files in proposals/left-proposals/ (exclude archive/ subdirectory)
    - For each file:
      - Parse YAML frontmatter to get created date and status
      - Skip if status is not "pending" (already processed)
      - Skip if created date is within autoArchiveDays of now
      - **Archive:** Read file content. Update frontmatter: set status: archived, add archived-at: ISO-8601. Write to proposals/left-proposals/archive/ with same filename. Delete original from proposals/left-proposals/.
    - Return: { archived, skipped }

    Both functions are designed to be called from scheduled tasks or on-demand. They are idempotent — safe to run multiple times.
