  <action>
    Create src/memory-proposals.js with exports: writeCandidate, readProposals, acquireLock, releaseLock, flushPendingBuffer, generateCandidateId.

    **generateCandidateId():**
    - Format: `mem-YYYYMMDD-NNN` per D-56
    - Date is current date. NNN is sequence number — read current file to find highest sequence for today, increment by 1. Start at 001 if first of the day.

    **acquireLock() / releaseLock():**
    - Lock file at `proposals/memory-proposals.md.lock` per D-58
    - acquireLock: attempt to create lock file exclusively (fs.writeFile with flag 'wx'). If file exists, retry every 500ms up to 5-second timeout. On timeout, return { acquired: false }.
    - releaseLock: remove lock file. Always succeeds (unlink with ignore-if-missing).
    - Lock file contains: `{ pid: process.pid, acquired: ISO-8601, holder: 'memory-extractor' }`

    **writeCandidate(candidate):**
    - candidate object: `{ content, category, sourceRef, confidence, rationale, sessionId, sourceFile, extractionTrigger }`
    - Compute content_hash: SHA-256 of normalized content (trimmed, lowercased), first 12 hex chars per D-56
    - **Dedup check per D-27/D-66:** Check content_hash against:
      1. All candidates in proposals/memory-proposals.md (any status)
      2. All entries in memory/memory.md
      3. All entries in memory-archive/*.md
      If match found: log "duplicate skipped", return { written: false, reason: 'duplicate' }
    - Acquire lock. If lock timeout: write candidate as JSON line to `proposals/memory-proposals-pending.jsonl` per D-58. Return { written: true, buffered: true }.
    - Generate candidate ID
    - Build section text per D-56:
      ```
      ### <candidate-id> . <CATEGORY> . <source-ref-short>
      - [ ] accept
      - [ ] reject
      - [ ] edit-then-accept
      - [ ] defer

      **Content:** <one-paragraph proposed memory entry>
      **Proposed tags:** <comma-separated>
      **Proposed related:** <wikilinks>

      session_id:: <sessionId>
      captured_at:: <ISO-8601>
      source_file:: <sourceFile>
      category:: <category>
      confidence:: <confidence>
      content_hash:: <content_hash>
      status:: pending
      extraction_trigger:: <extractionTrigger>
      ```
    - Read current file, prepend new section after frontmatter (newest first per D-55)
    - Update frontmatter: `last_updated`, increment `total_pending`
    - Write file via vaultWrite
    - Release lock
    - Return { written: true, candidateId }

    **readProposals():**
    - Read proposals/memory-proposals.md
    - Parse each `### mem-` section into objects with all inline Dataview fields
    - Return array of candidate objects
    - On missing file: return empty array

    **flushPendingBuffer():**
    - Read proposals/memory-proposals-pending.jsonl
    - For each line: parse JSON, call writeCandidate (which will re-acquire lock and dedup)
    - On success: truncate the .jsonl file
    - Called at start of any writeCandidate that successfully acquires lock

    Source-ref-short derivation: For `session:<id>`, short form is `session:<first 8 chars>`. For `file:<path>`, short form is filename without extension. For `daily:<date>`, short form is `daily:<date>`.

    Create src/memory-extractor.js with exports: extractFromTranscript, extractFromFile, extractFromDirectory, extractMemories.

    **extractFromTranscript(transcriptPath, sessionId, options = {}):**
    - Read JSONL file line-by-line (stream, not full load per D-44)
    - Filter messages per D-45:
      - Include: user messages, assistant text content, tool outputs that represent decisions/outcomes (git diff summaries, merged PR titles, created/modified file paths)
      - Exclude: system-reminder messages (role === 'system'), messages < 20 chars, raw Read/Glob/Grep/Bash-ls tool output (tool_name matches these and output is not a decision/outcome)
    - Weight git diff and PR content 2x per D-45 (include twice in extraction corpus or mark with [HIGH-SIGNAL] prefix)
    - Check size: if > pipeline.extraction.oversizeThresholdBytes or > pipeline.extraction.oversizeThresholdMessages:
      - Chunk into pipeline.extraction.chunkSize (100) message windows with pipeline.extraction.chunkOverlap (10) message overlap per D-46
      - Extract from each chunk independently
      - Deduplicate across chunks by content_hash before writing
    - For each chunk/corpus: call Haiku with extraction prompt:
      - System prompt: lists all 7 categories from templates.json memory-categories with description, example, exclusion rules. Includes D-24 exclusions (routine actions, status updates, debugging steps, TODO items, uninterpreted third-party quotes, ISPN/Genesys/Asana content, content already in memory.md). States: "OTHER requires one-sentence justification."
      - Expected JSON response: `[{ "category": "DECISION", "content": "...", "source_ref": "...", "confidence": 0.0-1.0, "rationale": "..." }]`
    - For each candidate:
      - If confidence < pipeline.extraction.confidenceLowConfidence (0.5): drop
      - If confidence >= 0.5: write via memory-proposals.writeCandidate with:
        - sourceRef: `session:<sessionId>` per D-65
        - sourceFile: transcriptPath
        - extractionTrigger: 'wrap'
        - sessionId: sessionId from hook stdin
    - On Haiku API failure: log warning, return empty array (never block /wrap per D-64)
    - Return: array of { candidateId, category, written } for each candidate processed

    **extractFromFile(filePath, options = {}):**
    - Read vault file content via vaultRead
    - Send to same Haiku extraction prompt
    - sourceRef: `file:<relative-vault-path>` per D-65
    - extractionTrigger: 'extract-memories'
    - sessionId: 'manual'

    **extractFromDirectory(dirPath, options = {}):**
    - List .md files in directory
    - Call extractFromFile for each
    - Return aggregated results

    **extractMemories(options = { file, dir, since, dailyRange }):**
    - Dispatches to appropriate extractor based on options:
      - `--file <path>`: extractFromFile
      - `--dir <path>`: extractFromDirectory
      - `--since <date>`: extractFromDirectory on Daily/ filtering by file date >= since
      - `--daily-range <start> <end>`: extractFromDirectory on Daily/ filtering date range
    - This is the `/extract-memories` command entry point per D-21 trigger (2)
