    Create the vault index portion of src/wikilink-engine.js.

    **buildVaultIndex():**
    - Scans all directories listed in vault-paths.json (both left and right)
    - For each .md file: reads first 5 lines, extracts:
      - `path`: relative vault path
      - `title`: filename without extension (or YAML title if present in frontmatter)
      - `firstLine`: first non-frontmatter, non-empty line of content (max 200 chars)
      - `tags`: array of tags from YAML frontmatter `tags:` field (empty array if none)
    - Excludes files in `proposals/` directory (proposals are transient, not linkable per D-18)
    - Writes result to `.cache/vault-index.json`
    - Creates `.cache/` directory if missing
    - Add `.cache/` to .gitignore if not already present

    **loadVaultIndex():**
    - Reads and parses `.cache/vault-index.json`
    - Returns empty array if file missing or corrupt (log warning, don't throw)

    **refreshIndexEntry(relativePath):**
    - Loads current index
    - Reads the file at relativePath, extracts same metadata as buildVaultIndex
    - If path exists in index: update entry
    - If path not in index: append entry
    - Writes updated index back to `.cache/vault-index.json`
    - This is called after vault-gateway write events per D-18 ("refresh on vault-gateway successful write")
  </action>

  <action>
    Add the suggestion pipeline to src/wikilink-engine.js per D-17/D-19:

    **suggestWikilinks(noteBody, noteTags, options = { isLeftProposal: false, correlationId }):**

    **Step 1 — Tokenize and extract (D-19 step 1):**
    - Split noteBody into tokens (words)
    - Extract proper nouns (capitalized words not at sentence start)
    - Pull tags from noteTags array

    **Step 2 — Cheap first pass: filename/title search (D-19 step 2):**
    - Load vault index via loadVaultIndex()
    - Score each index entry by:
      - Title overlap: count of shared tokens between input and entry.title (weight: 3)
      - FirstLine hits: count of shared tokens between input and entry.firstLine (weight: 1)
      - Tag overlap: count of shared tags between noteTags and entry.tags (weight: 2)
    - Sort by score descending, take top pipeline.wikilink.candidatePoolSize (20) candidates

    **Step 3 — Haiku re-rank (D-19 step 3):**
    - If zero candidates from Step 2: return empty result immediately
    - Send candidates to Haiku with prompt: "Given this note content, rank these candidate notes by relevance. For each, return a relevance score 0.0-1.0 and a 6-word reason for the connection."
    - Expected JSON: `[{ "path": "...", "relevance": 0.0-1.0, "reason": "6 word reason here" }]`
    - Filter by relevance >= pipeline.wikilink.relevanceThreshold (0.6)
    - Take top pipeline.wikilink.maxSuggestions (5) results
    - On Haiku failure: fall back to top 3 filename-match results without reasons (enrichment failure never blocks per D-39)

    **Step 4 — Format output (D-19 step 4):**
    - If options.isLeftProposal: format as `## Suggested wikilinks` section per D-20
    - Else: format as `## Related` section
    - Each link: `- [[Note Title]] — <6-word reason>` (or `- [[Note Title]]` if no reason from fallback)
    - If zero results meet threshold: return empty string (omit section entirely per D-17)

    Returns: `{ section: string, links: [{ path, title, relevance, reason }] }`
  </action>
