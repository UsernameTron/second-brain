  <action>
Create `src/today-command.js` exporting a single async function `runToday(options)`.

**Function signature:**
```javascript
async function runToday(options = {})
```
Where `options`:
- `mcpClient` — injected MCP client (required for connectors; null in dry-run/test)
- `mode` — `'interactive'` (default), `'scheduled'`, or `'dry-run'` (per D-05)
- `projectsDir` — override for `~/projects/` (for testing slippage scanner)
- `vaultRoot` — override for `~/Claude Cowork/` (for testing output)
- `date` — override for today's date (for testing)
- `haikuClient` — injected LLM client (for testing; created via `createHaikuClient()` if not provided)

**Internal structure — 6 functions plus orchestrator:**

1. **`_fanOut(mcpClient)`** — `Promise.allSettled` over `getCalendarEvents(mcpClient)`, `getRecentEmails(mcpClient, { vipOnly: true })`, `getGitHubActivity(mcpClient)`. Returns object `{ calendar, gmail, github }` where each value is the connector result. For settled-rejected promises (should not happen due to no-throw contract, but defensive), wrap in `{ success: false, data: null, error: rejection.reason, source, fetchedAt }`.

2. **`_getPipelineState()`** — calls `getProposalsPendingCount()` and `getDeadLetterSummary()`, returns `{ proposalCount, deadLetter }`. Wrapped in try/catch — on failure returns `{ proposalCount: 0, deadLetter: { pending: 0, frozen: 0, total: 0, warning: false } }` and sets pipeline source to degraded.

3. **`_scanSlippage(projectsDir, config)`** — the inline slippage scanner per D-12/D-13:
   - Glob `${projectsDir}/*/.planning/STATE.md` using `fs.readdirSync` on projectsDir then check each for `.planning/STATE.md`
   - For each found STATE.md, parse YAML frontmatter using `gray-matter`
   - Extract: `status`, `last_activity`, `progress.percent`, `progress.completed_phases`, `progress.total_phases`
   - Calculate days since `last_activity` compared to `date` parameter
   - Flag as stalled if days > `config.slippage.staleDays`
   - Skip projects in `config.slippage.excludeProjects`
   - Cap at `config.slippage.maxProjects`
   - Per D-14: if a STATE.md can't be parsed (malformed YAML, missing frontmatter), skip that project and add to `warnings` array
   - Returns `{ projects: [...], warnings: [] }` where each project is `{ name, status, phase, percent, daysSinceActivity, stalled }`

4. **`_identifyFrog(slippageData, haikuClient)`** — per D-15:
   - Build candidate list from slippage data (stalled projects, low-percent phases)
   - If no candidates, return `{ frog: null, reasoning: 'No stalled projects or overdue tasks found.' }`
   - Call `haikuClient.classify()` with system prompt: "You identify the single hardest or most-avoided task from a list of project statuses. Return JSON: {frog: string, reasoning: string}. The frog is the task the user is most likely avoiding or that will cause the most damage if delayed further."
   - User content: JSON.stringify of candidate list
   - On LLM failure (success===false), fall back to heuristic: pick the project with most days stalled
   - Returns `{ frog: string, reasoning: string }`

5. **`_buildSourceHealth(connectorResults, pipelineOk)`** — builds the D-08 frontmatter `sources` map:
   - For each connector result: `ok` if `success===true`, `degraded` if `success===false`
   - Pipeline: `ok` if pipelineOk, `degraded` otherwise
   - Returns `{ sources: { calendar, gmail, github, pipeline }, degradedCount }`

6. **`_renderBriefing(data)`** — renders the full D-09 markdown template:
   - `data` contains: `{ date, sourceHealth, connectorResults, pipelineState, slippage, frog, mode }`
   - Build frontmatter per D-08: `date`, `sources` map, `degraded` count, `generated` ISO timestamp, `mode`
   - Heading: `# Daily Briefing -- DayOfWeek, Mon DD` (using `date-fns` `format()` if available, else manual)
   - Synthesis blockquote: if ALL sources degraded, render diagnostic checklist per D-11. Otherwise, call `haikuClient.classify()` with prompt: "Write a 3-4 sentence synthesis of today's briefing data. Flag any degraded sources. Be direct and concise." with all section data as context. On LLM failure, render static synthesis: "> Briefing generated with [N] degraded source(s)."
   - Horizontal rule after synthesis per D-07
   - 6 sections in order per D-03, using human-facing names per D-07:
     - **## Meetings** — bullet list of calendar events (time, title, attendees). If degraded: `[calendar] {error}` per D-10
     - **## VIP Emails** — subject, sender, snippet. If degraded: `[gmail] {error}` per D-10
     - **## Slippage** — stalled projects with name, phase, percent, days stalled. Wikilinks per D-06 (only to vault notes that exist). If no slippage: "No stalled projects."
     - **## Frog** — single task with reasoning from `_identifyFrog`. If null: "No frog identified today."
     - **## GitHub** — commits/PRs/issues per repo. If degraded: `[github] {error}` per D-10
     - **## Pipeline** — use `formatBriefingSection('proposals', { count })` and `formatBriefingSection('deadletter', deadLetter)`. If both empty: "Pipeline clear."
   - Error format per D-10: degraded sections render as a warning emoji followed by `[SOURCE] ERROR_CODE: human message` where the error string matches `/^[A-Z_]+: .+/` — pull the error code from the connector's error field, or use `MCP_ERROR` as fallback code

7. **`runToday(options)` orchestrator:**
   - Load config via `loadPipelineConfig()`
   - Set defaults: `projectsDir` = `~/projects/`, `vaultRoot` = `~/Claude Cowork/`, `date` = today, `mode` = `'interactive'`
   - Create `haikuClient` via `createHaikuClient()` if not injected
   - Run fan-out, pipeline state, and slippage scan in parallel (`Promise.allSettled`)
   - Identify frog from slippage data
   - Build source health
   - Render briefing markdown
   - Determine output path per D-02/D-05:
     - `dry-run`: `${vaultRoot}/RIGHT/daily/_dry-run-YYYY-MM-DD.md`
     - Normal: `${vaultRoot}/RIGHT/daily/YYYY-MM-DD.md`
   - Write daily note using `fs.promises.writeFile` (create `RIGHT/daily/` directory with `{ recursive: true }` if needed)
   - If `mode === 'interactive'`, also print briefing to stdout via `console.log`
   - Return `{ path: outputPath, briefing: markdownString, sourceHealth }`
