<objective>
Create the project skeleton and write-gateway module with canonical path enforcement, config-driven vault bootstrapping, redacted quarantine, and wikilink utilities.

Purpose: Establishes the single enforcement point that every future vault write must pass through. Without this, subsequent phases have no mechanism to prevent writes to LEFT-side paths or unknown paths. This plan creates the foundation that Plan 02 extends with content filtering and style linting.

Output: Working `src/vault-gateway.js` with hardened path guard (canonical resolution, absolute path rejection, symlink defense), redacted quarantine to `proposals/`, config hot-reload, config-driven bootstrap, and comprehensive tests. RIGHT-side vault directories created.

**Review-driven changes from original plan:**
- Path security hardened: canonical resolution via `path.resolve` against vault root, absolute path rejection, symlink check via `fs.realpathSync`
- Quarantine stores redacted metadata only (reason, original path, timestamp) — no blocked content written to disk
- Bootstrap derives directories from `config.right` array instead of hardcoding 9 directory names
- Wikilink generation includes path-qualified variant `[[Folder/Note]]` for disambiguation
- Added config schema validation at load time (fail closed on invalid config)

- Added case-variant path test confirming BLOCK on case-insensitive FS (Codex+Claude review item #2)

    Create `src/vault-gateway.js` as a CommonJS module (`module.exports = { ... }`). The module implements:

    **Constants:**
    - `VAULT_ROOT = process.env.VAULT_ROOT || path.join(process.env.HOME, 'Claude Cowork')` — vault path with space, per PROJECT.md. Overridable via VAULT_ROOT env var for integration testing (addresses review item #6).
    - `CONFIG_DIR` = resolved path to `config/` directory relative to project root

    **Config loading, validation, and hot-reload (per D-05, D-09, research Pattern 2):**
    - `loadConfig()` — reads `vault-paths.json` and `excluded-terms.json`. Validates schema: `left` and `right` must be non-empty arrays of strings, `excludedTerms` must be a non-empty array of strings. **LEFT/RIGHT intersection check (addresses review item #3):** if any entry appears in both `left` and `right` arrays, throw with message `"Config error: '${entry}' appears in both left and right arrays — ambiguous boundary"`. On any validation failure, throw with descriptive message (fail closed at startup). Returns `{ left: string[], right: string[], excludedTerms: string[] }`.
    - `getConfig()` — returns cached config, lazy-loads on first call
    - `watchConfig()` — uses `fs.watch` with `{ persistent: false }` on both config files. Debounce 50ms to prevent macOS double-fire (research Pitfall 2). On parse error during hot-reload, keep old config and log warning to stderr.
    - `initGateway()` — loads config, starts watchers, returns config. Called once at module load or explicitly.

    **Canonical path security (addresses review HIGH concern #1):**

    `normalizePath(inputPath)` must enforce ALL of these:
    1. **Reject absolute paths:** If `inputPath` starts with `/` or matches `path.isAbsolute()`, throw `VaultWriteError` with code `INVALID_PATH` and message "Absolute paths are not allowed".
    2. **Normalize:** Apply `path.normalize(inputPath)` to resolve `.` and `..` segments.
    3. **Reject traversal:** After normalize, if result contains `..` (check with `normalized.includes('..')` or `normalized.startsWith('..')`) throw `VaultWriteError` with code `INVALID_PATH` and message "Path traversal detected".
    4. **Resolve against vault root:** Compute `resolvedPath = path.resolve(VAULT_ROOT, normalized)`. Verify `resolvedPath.startsWith(VAULT_ROOT + path.sep)` or `resolvedPath === VAULT_ROOT`. If not, throw `VaultWriteError` with code `INVALID_PATH` and message "Path escapes vault root".
    5. **Symlink defense (write operations only):** In `vaultWrite`, after the path passes all checks, before writing, call `fs.realpathSync(path.dirname(resolvedPath))` on the parent directory (if it exists) and verify the real path is still inside VAULT_ROOT. If not, throw with code `INVALID_PATH` and message "Symlink escape detected". If the parent directory does not yet exist (will be created by mkdir), skip symlink check since there is nothing to resolve yet.

    Return the normalized relative path (for use in allowlist matching).

    **Path guard (per D-01, D-04):**
    - `checkPath(normalizedPath, config)` — extracts the first path segment (everything before the first `/`, or the whole string if no `/`). Checks if it matches any entry in `config.right`. Returns `{ decision: 'PASS' }` or `{ decision: 'BLOCK', reason: "Path 'X' is not on the RIGHT side allowlist" }`. Uses case-sensitive string comparison. **Case-insensitive filesystem note (addresses review item #2):** macOS APFS defaults to case-insensitive (case-preserving). This means `Memory/note.md` resolves to the same file as `memory/note.md` on disk, but checkPath uses case-sensitive matching against the allowlist. This is intentionally fail-safe: a case-variant like `Memory/note.md` is BLOCKED (does not match `memory` in config.right), preventing false allows. A case-variant path will never accidentally gain write access — it will only be blocked. Add an inline comment in the implementation: `// Case-sensitive match: fail-safe on case-insensitive FS (blocks 'Memory/' even though 'memory/' is allowed)`.

    **Vault read guard (per D-04 three-tier model):**
    - `vaultRead(relativePath)` — normalizes path (using normalizePath, rejecting absolute paths and traversal), checks if first segment is in LEFT or RIGHT list. If in neither, throws `VaultWriteError` with code `PATH_BLOCKED` ("Path not in vault manifest — unknown paths blocked per D-04"). If path is valid, reads and returns file content via `fs.promises.readFile`.

    **Redacted quarantine (addresses review HIGH concern #2 — "excluded content never reaches disk"):**

    `quarantine(originalPath, reason)` — note: does NOT accept `content` parameter. Writes a metadata-only file to `proposals/quarantine-{ISO-timestamp}.md` in the vault. File contains ONLY:
    ```
    ---
    quarantine: true
    original_path: {originalPath}
    reason: {reason}
    timestamp: {ISO timestamp}
    ---

    Content was blocked and not written to disk.
    Review the source and re-submit manually if appropriate.
    ```
    No blocked content is stored. This satisfies the phase goal "excluded content never reaches disk." Returns `{ decision: 'QUARANTINED', quarantinePath }`.

    **Core write function (partial — path guard only in this plan):**
    - `vaultWrite(relativePath, content, options = {})`:
      1. Call `normalizePath(relativePath)` — rejects absolute, traversal, escapes
      2. Call `checkPath(normalized, config)` — if BLOCK, throw `VaultWriteError` with code `PATH_BLOCKED`
      3. Compute `absolutePath = path.join(VAULT_ROOT, normalized)`
      4. Create parent directory: `await fs.promises.mkdir(path.dirname(absolutePath), { recursive: true })`
      5. Symlink defense: if parent directory existed before mkdir, call `fs.realpathSync(path.dirname(absolutePath))` and verify it starts with VAULT_ROOT. Throw `INVALID_PATH` if not.
      6. Write: `await fs.promises.writeFile(absolutePath, content, 'utf8')`
      7. Return `{ decision: 'WRITTEN', path: normalized }`
    - NOTE: Content filter (Guard 2) and style lint (Guard 3) are added in Plan 02. This plan implements the path guard (Guard 1) only, plus the quarantine mechanism that Guards 2 and 3 will use.

    **Config-driven vault bootstrap (addresses review MEDIUM concern — "derive from config, not hardcode"):**
    - `bootstrapVault()` — reads `config.right` array from `getConfig()` and creates each directory under VAULT_ROOT using `fs.promises.mkdir({ recursive: true })`. Does NOT hardcode directory names. Does NOT create LEFT-side directories (those are human-created per VAULT-01).

    **Wikilink utility (per D-16, XREF-01, addresses review concern about disambiguation):**
    - `toWikilink(noteName, displayText)` — returns `[[noteName|displayText]]` if displayText provided, else `[[noteName]]`. Standard Obsidian wikilink syntax.
    - `toQualifiedWikilink(folder, noteName, displayText)` — returns `[[folder/noteName|displayText]]` or `[[folder/noteName]]`. Use when note names are ambiguous across directories (Obsidian resolves path-qualified links deterministically).

    **Audit logging (addresses Codex review: observability gap):**
    - `logDecision(action, path, decision, reason)` — writes a structured JSON line to stderr via `console.error(JSON.stringify({ ts: new Date().toISOString(), action, path, decision, reason }))`. Called at every decision point: WRITTEN, BLOCKED, QUARANTINED, STYLE_VIOLATION, CONTENT_BLOCKED. **No blocked content is included in the log** — only the decision metadata (path, decision code, reason string). This provides rollout observability without storing sensitive content.

    **Error class:**
    - `class VaultWriteError extends Error` with `code` property. Codes: `INVALID_PATH`, `PATH_BLOCKED`, `STYLE_VIOLATION` (used in Plan 02), `CONTENT_BLOCKED` (used in Plan 02).
