  <action>
    Create `src/vault-gateway.js` as a CommonJS module (`module.exports = { ... }`). The module implements:

    **Constants:**
    - `VAULT_ROOT = path.join(process.env.HOME, 'Claude Cowork')` — vault path with space, per PROJECT.md
    - `CONFIG_DIR` = resolved path to `config/` directory relative to project root

    **Config loading and hot-reload (per D-05, D-09, research Pattern 2):**
    - `loadConfig()` — reads `vault-paths.json` and `excluded-terms.json`, returns `{ left: string[], right: string[], excludedTerms: string[] }`
    - `getConfig()` — returns cached config, lazy-loads on first call
    - `watchConfig()` — uses `fs.watch` with `{ persistent: false }` on both config files. Debounce 50ms to prevent macOS double-fire (research Pitfall 2). On parse error, keep old config and log warning.
    - `initGateway()` — loads config, starts watchers, returns config. Called once at module load or explicitly.

    **Path normalization (research Pitfall 1 — path traversal prevention):**
    - `normalizePath(inputPath)` — calls `path.normalize()`, strips leading `/`, rejects any path still containing `..` by throwing `VaultWriteError` with code `INVALID_PATH`

    **Path guard (per D-01, D-04):**
    - `checkPath(normalizedPath, config)` — checks if the first path segment matches any entry in `config.right`. Returns `{ decision: 'PASS' }` or `{ decision: 'BLOCK', reason: "Path 'X' is not on the RIGHT side allowlist" }`. Uses case-sensitive matching (Obsidian paths are case-sensitive on macOS APFS).

    **Vault read guard (per D-04 three-tier model):**
    - `vaultRead(relativePath)` — normalizes path, checks if first segment is in LEFT or RIGHT list. If in neither, throws `VaultWriteError` with code `PATH_BLOCKED` ("Path not in vault manifest"). If path is valid, reads and returns file content via `fs.promises.readFile`.

    **Quarantine (per D-08, research Pattern 3):**
    - `quarantine(originalPath, content, reason)` — writes to `proposals/quarantine-{ISO-timestamp}.md` in the vault. Frontmatter: `quarantine: true`, `original_path`, `reason`, `timestamp`. Strips existing frontmatter from content before embedding (research Pitfall 5). Returns `{ decision: 'QUARANTINED', quarantinePath }`.

    **Core write function (partial — path guard only in this plan):**
    - `vaultWrite(relativePath, content, options = {})` — normalizes path, runs `checkPath`. If BLOCK, throws `VaultWriteError` with code `PATH_BLOCKED`. If PASS, writes to `path.join(VAULT_ROOT, normalized)` via `fs.promises.writeFile` with `{ encoding: 'utf8' }`, creating parent directories with `fs.promises.mkdir({ recursive: true })`. Returns `{ decision: 'WRITTEN', path: normalized }`.
    - NOTE: Content filter (Guard 2) and style lint (Guard 3) are added in Plan 02. This plan implements the path guard (Guard 1) only, plus the quarantine mechanism that Guards 2 and 3 will use.

    **Vault bootstrap (per VAULT-01, research Pattern 4):**
    - `bootstrapVault()` — creates all 9 RIGHT-side directories under VAULT_ROOT: `memory`, `briefings`, `ctg`, `job-hunt`, `interview-prep`, `content`, `research`, `ideas`, `proposals`. Uses `fs.promises.mkdir({ recursive: true })`. Does NOT create LEFT-side directories (those are human-created).

    **Wikilink utility (per D-16, XREF-01):**
    - `toWikilink(noteName, displayText)` — returns `[[noteName|displayText]]` if displayText provided, else `[[noteName]]`. Standard Obsidian wikilink syntax. Obsidian resolves these across the entire vault regardless of directory.

    **Error class:**
    - `class VaultWriteError extends Error` with `code` property. Codes: `INVALID_PATH`, `PATH_BLOCKED`, `STYLE_VIOLATION` (used in Plan 02).

    **Exports:**
    ```javascript
    module.exports = {
      vaultWrite, vaultRead, toWikilink, bootstrapVault,
      initGateway, getConfig, normalizePath, checkPath, quarantine,
      VaultWriteError, VAULT_ROOT
    };
    ```
