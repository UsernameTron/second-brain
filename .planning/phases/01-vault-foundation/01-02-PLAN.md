    **Create `src/content-policy.js` as a CommonJS module:**

    This module handles Guard 2 (content filtering) per D-06 through D-11, with sanitization per VAULT-03.

    Import `escapeRegex` from `../src/utils.js` (created in Plan 01, addresses review item #5). Do NOT define escapeRegex in this module — use `const { escapeRegex } = require('./utils');`.

    `checkContent(content, excludedTerms)` async function:
    - Stage 1: Keyword scan. For each term in excludedTerms, build regex `new RegExp('\\b' + escapeRegex(term) + '\\b', 'i')`. Test against content. No match = return `{ decision: 'PASS' }` immediately (zero-latency common case per D-06).
    - Stage 2 (fires only on Stage 1 match): Call `classifyWithHaiku(content, matchedTerm, excludedTerms)`.
    - Return values:
      - No keyword match: `{ decision: 'PASS' }`
      - Keyword match + Haiku says BLOCK: `{ decision: 'BLOCK', reason: 'Excluded content detected (term: X)', matchedTerm: 'X' }`
      - Keyword match + Haiku says ALLOW: `{ decision: 'PASS' }`
      - Keyword match + Haiku error/timeout: `{ decision: 'BLOCK', reason: 'Haiku unavailable, queued for review (error message)', matchedTerm: 'X' }`

    `sanitizeContent(content, excludedTerms)` function (addresses agreed HIGH concern #1 — contamination radius):
    - Attempts to remove excluded references from content while preserving the rest.
    - **Paragraph-level stripping with contamination radius** (NOT sentence-level — sentence-level misses adjacent confidential context):
      1. Split content into paragraphs (split on `\n\n` — double newline).
      2. For each paragraph, check if any excluded term appears (case-insensitive, word-boundary match using escapeRegex from utils.js).
      3. If a paragraph contains ANY excluded term, replace the ENTIRE paragraph with `[REDACTED]`. This is the "contamination radius" — adjacent sentences in the same paragraph are assumed to share context with the keyword and are stripped together. Example: "The routing architecture at Genesys handles 50,000 sessions. Each session uses a proprietary load-balancing algorithm." Both sentences are in one paragraph, both get redacted even though only the first contains the keyword.
      4. Paragraphs with no excluded terms are preserved unchanged.
      5. Reassemble with `\n\n` between paragraphs.
    - Returns `{ sanitized: string, redactedCount: number, markers: string[] }` where `redactedCount` is the number of paragraphs replaced and `markers` is an array of `[REDACTED]` placeholder descriptions.
    - **[REDACTED] marker behavior (addresses review item #8):** Each redacted paragraph is replaced with the literal string `[REDACTED]` on its own line. No information about what was redacted is included in the marker. The quarantine metadata file (from Plan 01) records the reason and original path — that is the only place the redaction reason is stored. Consumers of sanitized notes should treat `[REDACTED]` as "content was removed for compliance reasons — check proposals/ for the quarantine record."
    - **Threshold:** If more than 50% of paragraphs are redacted, the content is primarily about excluded topics. In this case, sanitization is considered insufficient — the gateway will quarantine instead of writing the sanitized version. This threshold operates on paragraph count, not character count, to avoid gaming by padding with short paragraphs.

    `sanitizeTermForPrompt(term)` function (addresses Gemini review MEDIUM — prompt injection via excluded terms):
    - Sanitizes an excluded-terms value before interpolation into the Haiku classifier system prompt.
    - Steps:
      1. Strip newlines and carriage returns (`\n`, `\r`) — prevents multi-line injection.
      2. Truncate to 50 characters — no legitimate excluded term needs more.
      3. Reject terms matching instruction-like patterns: `/^(you must|always|never|ignore|respond with|say |forget|disregard|override)/i` — log a warning to stderr (`console.error(\`[content-policy] Skipping suspicious excluded term: "${term}"\`)`) and return `null`.
      4. Strip leading/trailing whitespace.
    - Returns the sanitized string, or `null` if the term was rejected.
    - Used by `classifyWithHaiku` to sanitize all terms before prompt interpolation. Null values are filtered out.

    `classifyWithHaiku(content, matchedTerm, excludedTerms, contextChars)` async function:
    - **Prompt injection defense (Gemini review MEDIUM):** Before interpolating excludedTerms into the system prompt, map each term through `sanitizeTermForPrompt()` and filter out nulls. The sanitized array is used in the system prompt; the original array is never interpolated directly. Also sanitize `matchedTerm` before interpolating into the user message.
    - **Privacy-preserving configurable context window (addresses review items #4 and round 2 #4):** Instead of sending full content, extract a window of `contextChars` characters (default 100, configurable) before and after each keyword match. The `contextChars` value is read from `config/vault-paths.json` field `haikuContextChars` (optional, default 100 if absent). Send only these context windows to Haiku, not the full note body. Format: `"...{N chars before}[MATCHED_TERM]{N chars after}..."` for each match (max 3 windows).
    - Uses `@anthropic-ai/sdk`: `const Anthropic = require('@anthropic-ai/sdk'); const client = new Anthropic();`
    - Calls `client.messages.create()` with:
      - `model: 'claude-haiku-4-5'`
      - `max_tokens: 10`
      - `timeout: 2000` (2s per D-08)
      - System prompt with BLOCK/ALLOW examples per research Pitfall 3:
        ```
        You are a content filter. Classify the following content excerpt as BLOCK or ALLOW.
        BLOCK: content that is about or from these organizations — their architecture, internal processes, client data, confidential strategy, or internal people: ${sanitizedTerms.join(', ')}
        ALLOW: neutral tool references ("tracked in Asana"), career narrative ("led a team at Genesys"), generic industry mentions.

        BLOCK examples:
        - "The Genesys Cloud routing configuration for ISPN queue handling..." (architecture/operations)
        - "Asana project ID 12345 with client Acme..." (client data)

        ALLOW examples:
        - "I led a 12-person contact center team at Genesys." (career narrative)
        - "Tracked in Asana" (neutral tool reference)
        - "Genesys is a common CCaaS vendor in the enterprise market." (generic industry mention)

        Respond with only the word BLOCK or ALLOW.
        ```
      - User message: `"Classify these excerpts around the term '${matchedTerm}':\n\n${contextWindows.join('\n\n')}"` — never the full note body.
    - Parse response: if `response.content[0].text.trim().toUpperCase() === 'BLOCK'` return `'BLOCK'`, else return `'ALLOW'`.
    - On any error (timeout, API down, malformed response): throw the error. The caller catches and returns BLOCK.

    Exports: `module.exports = { checkContent, classifyWithHaiku, sanitizeContent, sanitizeTermForPrompt };` (escapeRegex is NOT exported from here — it lives in utils.js)

This is the mechanism that enforces D-12 — every vault-writing agent calls createVaultWriter and uses the returned prefix in its system prompt.
      - `write(relativePath, content, options)` — calls through to the gateway's `vaultWrite` with the agentName recorded for audit. Same signature as vaultWrite.
    - **Enforcement mechanism:** Future phases (Phase 2+) that generate vault content MUST use `createVaultWriter('agentName')` instead of calling `vaultWrite` directly. The wrapper ensures the style guide is available. Direct `vaultWrite` calls still work (for non-generated content), but `createVaultWriter` is the required entry point for LLM-generated output.
