    **IMPORTANT — Verified MCP_DOCKER GitHub tool surface (2026-04-22):**
    The Docker MCP GitHub server does NOT expose a `list_user_events` tool. The actual per-repo tools are:
    - `list_commits` — params: `{owner, repo, since?, until?, author?, perPage?}`
    - `list_issues` — params: `{owner, repo, since?, state?, perPage?}`
    - `list_pull_requests` — params: `{owner, repo, state?, perPage?}`

    The connector must compose activity from these three tools per configured repo.

    Implementation:

    1. Config uses lazy memoized loader (see 03-01 patch):
       ```js
       let _config = null;
       function getConfig() { if (!_config) _config = loadConnectorsConfig().github; return _config; }
       ```

    2. `getGitHubActivity(mcpClient, options = {})`:
       - Guard: if `!mcpClient` return `makeError(SOURCE.GITHUB, 'mcpClient is required')`
       - `const config = getConfig()`
       - `const owner = config.owner` (default: 'UsernameTron')
       - `windowHours = options.hours || config.defaultWindowHours`
       - Compute `since` as ISO8601 string (now - windowHours)
       - For each repo in `config.repos`, call three MCP tools inside try/catch:
         a. `mcpClient.callTool('list_commits', { owner, repo, since, perPage: 30 })`
         b. `mcpClient.callTool('list_issues', { owner, repo, since, state: 'all', perPage: 30 })`
         c. `mcpClient.callTool('list_pull_requests', { owner, repo, state: 'all', perPage: 30 })`
       - Use `Promise.allSettled` to gather results across repos and tool types
       - Filter PRs to only those updated within the window (list_pull_requests lacks `since` param — filter client-side by `updated_at`)
       - Combine into: `{ commits: [...], issues: [...], pullRequests: [...] }`
       - Return `makeResult(SOURCE.GITHUB, combinedData)`
       - If ALL calls fail: return `makeError(SOURCE.GITHUB, 'All GitHub MCP calls failed')`
       - If SOME calls fail: return `makeResult` with partial data (non-failing calls included), add `warnings` array to data

    3. Exports: `{ getGitHubActivity }`
