    Per D-11: The connector calls Docker MCP tools (`mcp__MCP_DOCKER__*`). Same dependency injection pattern — `mcpClient` parameter with `callTool(toolName, params)`.

    Per D-12: Repo scoping to UsernameTron happens inside the connector. No consumer-side filtering.

    Implementation:

    1. Load config once at module level: `const config = loadConnectorsConfig().github;`

    2. `getGitHubActivity(mcpClient, options = {})`:
       - `windowHours = options.hours || config.defaultWindowHours`
       - The Docker MCP GitHub server exposes tools for listing user events. Call `mcpClient.callTool('list_user_events', { username: 'UsernameTron' })` inside try/catch. (The username is the GitHub account, hardcoded to match D-12's "scoped to UsernameTron repos only".)
       - On success, filter the returned events array:
         a. Keep only events where `event.repo.name` starts with `UsernameTron/` or matches a repo in `config.repos` (comparison: `config.repos.some(r => event.repo.name === 'UsernameTron/' + r || event.repo.name === r)`)
         b. Keep only events where `event.type` is in `config.eventTypes`
         c. Keep only events where `event.created_at` is within the windowHours
       - Return `makeResult(SOURCE.GITHUB, { events: filteredEvents })`
       - On error: return `makeError(SOURCE.GITHUB, error.message || 'GitHub MCP call failed')`

    3. Exports: `{ getGitHubActivity }`
