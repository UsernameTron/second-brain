    Per D-04: The Gmail MCP server (`gmail-mcp-pete`) is a standalone repo. This connector calls it via MCP transport, same dependency injection pattern as calendar.

    Per D-08: The server exposes exactly 3 tools: `list_recent_messages`, `get_message_body`, `create_draft`. This connector maps to those three tools.

    Per D-09 + D-10: Two-layer VIP filtering. Server-side filter is pre-applied (the server only returns VIP messages). Connector-side filter (D-10) additionally narrows to the project-specific VIP list from `config/connectors.json`.

    Implementation:

    1. Load config once at module level: `const config = loadConnectorsConfig().gmail;`

    2. `getRecentEmails(mcpClient, options = {})`:
       - `windowHours = options.hours || config.defaultWindowHours`
       - `vipOnly = options.vipOnly !== undefined ? options.vipOnly : true`
       - `maxResults = options.maxResults || config.maxResults`
       - Call `mcpClient.callTool('list_recent_messages', { hours: windowHours, maxResults })` inside try/catch
       - On success:
         a. If `vipOnly === true`, filter messages to only those where `message.from` (email address portion) matches an entry in `config.vipSenders` (case-insensitive comparison)
         b. Return `makeResult(SOURCE.GMAIL, { messages: filteredMessages })`
       - On error: return `makeError(SOURCE.GMAIL, error.message || 'Gmail MCP call failed')`

    3. `getEmailBody(mcpClient, messageId)`:
       - Call `mcpClient.callTool('get_message_body', { messageId })` inside try/catch
       - On success: return `makeResult(SOURCE.GMAIL, { message: result })`
       - On error: return `makeError(SOURCE.GMAIL, error.message || 'Gmail message fetch failed')`

    4. `createDraft(mcpClient, { to, subject, body })`:
       - Call `mcpClient.callTool('create_draft', { to, subject, body })` inside try/catch
       - On success: return `makeResult(SOURCE.GMAIL, { draft: result })`
       - On error: return `makeError(SOURCE.GMAIL, error.message || 'Gmail draft creation failed')`
