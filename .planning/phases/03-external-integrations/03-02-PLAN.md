    Create `src/connectors/calendar.js` (CommonJS, 'use strict'):

    Per D-01: The connector calls Cowork native MCP tools. In Claude Code, MCP tools are called via the `mcp__claude_ai_Google_Calendar__*` tool namespace. However, connectors run as Node.js modules, not inside Claude Code's runtime. The connector must accept an MCP transport/client as a dependency injection parameter so tests can mock it and Claude Code can pass the real transport at call time.

    **Pattern:**
    ```js
    async function getCalendarEvents(mcpClient, options = {}) {
      // mcpClient.callTool('list_calendar_events', params) or equivalent
    }
    ```

    The `mcpClient` parameter is an object with a `callTool(toolName, params)` async method. In production, the Phase 4 `/today` command will construct this from Claude Code's MCP runtime. In tests, it's a Jest mock.

    Implementation:

    1. Load config once at module level: `const config = loadConnectorsConfig().calendar;`

    2. `getCalendarEvents(mcpClient, options = {})`:
       - Guard: if `!mcpClient` return `makeError(SOURCE.CALENDAR, 'mcpClient is required')`
       - `windowHours = options.hours || config.defaultWindowHours`
       - Compute `timeMin` (now) and `timeMax` (now + windowHours) as ISO8601 strings
       - Call `mcpClient.callTool('list_calendar_events', { timeMin, timeMax })` inside try/catch
       - On success, filter the returned events array:
         a. If `config.excludeDeclined === true`, remove events where attendee response for Pete's email is `'declined'`. Match attendee email case-insensitively against `config.userEmail || 'cpeteconnor@gmail.com'`
         b. For working-hours filtering: if event has `start.dateTime`, parse the hour in local timezone (`config.timezone` or system default via `Intl.DateTimeFormat().resolvedOptions().timeZone`). Remove events where start hour < `config.workingHours.start` OR >= `config.workingHours.end`. If event has only `start.date` (all-day event), skip working-hours filter (always include unless declined)
       - Return `makeResult(SOURCE.CALENDAR, { events: filteredEvents })`
       - On any error: return `makeError(SOURCE.CALENDAR, error.message || 'Calendar MCP call failed')`

    3. `getEvent(mcpClient, eventId)`:
       - Guard: if `!mcpClient` return `makeError(SOURCE.CALENDAR, 'mcpClient is required')`
       - Guard: if `!eventId` return `makeError(SOURCE.CALENDAR, 'eventId is required')`
       - Call `mcpClient.callTool('get_calendar_event', { eventId })` inside try/catch
       - On success: return `makeResult(SOURCE.CALENDAR, { event: result })`
       - On error: return `makeError(SOURCE.CALENDAR, error.message || 'Calendar event fetch failed')`

    4. Exports: `{ getCalendarEvents, getEvent }` — NO create/update/delete/patch functions per D-02.
