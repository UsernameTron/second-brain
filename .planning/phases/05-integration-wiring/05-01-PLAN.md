  <action>
    Replace the three stub functions in `~/projects/gmail-mcp-pete/src/tools.js` with real googleapis calls. The return shape must remain `{ content: [{ type: 'text', text: JSON.stringify(...) }] }` to maintain MCP protocol compatibility.

    **listRecentMessages(args)**:
    - Call `getAuthenticatedClient()` from `./auth-helper`
    - Build a `google.gmail('v5')` client (or current version — use `googleapis.google.gmail({ version: 'v1', auth: oAuth2Client })`)
    - Build Gmail query string from `allowedSenders`: if provided, join as `from:addr1 OR from:addr2`; if `args.hours` is provided, add `after:${epochSeconds}` to the query (per D-04, the connector passes `hours`)
    - Call `gmail.users.messages.list({ userId: 'me', q: query, maxResults: args.maxResults || 10 })`
    - For each message ID returned, call `gmail.users.messages.get({ userId: 'me', id: msgId, format: 'metadata', metadataHeaders: ['From', 'Subject', 'Date'] })`
    - Return array of `{ id, from, subject, date, snippet }` objects wrapped in MCP response shape
    - On error: return `{ content: [{ type: 'text', text: JSON.stringify({ error: true, message: err.message }) }] }`

    **getMessageBody(args)**:
    - Call `getAuthenticatedClient()`
    - Call `gmail.users.messages.get({ userId: 'me', id: args.messageId, format: 'full' })`
    - Decode the body: check `payload.parts` for `text/plain` MIME type, base64url-decode the `body.data` field using `Buffer.from(data, 'base64url').toString('utf8')` (Node 16+ supports base64url encoding natively, but safer to use `Buffer.from(data.replace(/-/g, '+').replace(/_/g, '/'), 'base64').toString('utf8')` for compatibility)
    - If no text/plain part, check `payload.body.data` directly
    - Return `{ messageId, body, from, subject, date }` wrapped in MCP response shape
    - On error: return error shape as above

    **createDraft(args)**:
    - Call `getAuthenticatedClient()`
    - Build RFC 2822 message: `To: ${args.to}\nSubject: ${args.subject}\n\n${args.body}`
    - Base64url-encode: `Buffer.from(rawMessage).toString('base64url')` (or manual replace for compat)
    - Call `gmail.users.drafts.create({ userId: 'me', requestBody: { message: { raw: encoded } } })`
    - Return `{ draftId: result.data.id, to: args.to, subject: args.subject }` wrapped in MCP response shape
    - NEVER call `gmail.users.messages.send` — draft-only per D-05

    All three functions must catch errors and return error responses (not throw), so the MCP server never crashes on a tool call failure.

    Add `hours` to the `list_recent_messages` tool schema in `index.js` if not already present (the connector passes it — see `src/connectors/gmail.js` line 104: `hours: windowHours`). Add to the inputSchema properties: `hours: { type: 'number', description: 'Only return messages from the last N hours' }`.
  </action>
