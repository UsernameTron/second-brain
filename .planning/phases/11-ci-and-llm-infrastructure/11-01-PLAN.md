Modify `createLlmClient()` in `src/pipeline-infra.js` per D-06 (config-driven endpoint swap, ~20-line change):

1. At the top of `createLlmClient()`, load pipeline config to check for local provider:
   ```javascript
   let llmConfig;
   try {
     const pipelineConfig = loadPipelineConfig();
     llmConfig = pipelineConfig.classifier && pipelineConfig.classifier.llm;
   } catch { llmConfig = null; }
   const useLocal = llmConfig && llmConfig.provider === 'local';
   ```

2. When `useLocal` is true, implement a `classifyLocal()` function that:
   - Uses Node.js built-in `fetch()` (available Node 18+) to POST to `${llmConfig.localEndpoint}/v1/chat/completions` (per D-08: OpenAI-compatible API format)
   - Request body: `{ model: llmConfig.localModel, messages: [{ role: "system", content: systemPrompt }, { role: "user", content: userContent }], max_tokens: maxTokens }`
   - Parses `response.choices[0].message.content` as JSON
   - Returns `{ success: true, data: parsedJSON }` on success
   - On connection error (fetch throws): log fallback via `logDecision('LLM_CLASSIFY', llmConfig.localModel, 'FALLBACK', 'local endpoint unreachable, falling back to Anthropic')`, then call the existing Anthropic-based `classify()` as fallback (per D-10)
   - On JSON parse error: return `{ success: false, error: ..., failureMode: 'parse-error' }` (same shape as Anthropic path)

3. The returned object's `classify` method should dispatch to `classifyLocal()` when `useLocal` is true, otherwise use the existing Anthropic `classify()`.

4. Do NOT add `openai` as a dependency (per D-09). Raw `fetch` only.

5. Do NOT modify `createHaikuClient()` or `createSonnetClient()` signatures. They still call `createLlmClient()` which now internally checks config.
