The skill frontmatter must have these exact field values:
- `name`: `pipeline-health`
- `description`: `Checks operational health of the second-brain pipeline — connector reachability, config validity, vault REST API status, and scheduler state. Use when diagnosing pipeline issues, after config changes, or for routine health checks. Trigger phrases: pipeline health, pipeline status, system health, connector status.`

The skill body must instruct Claude to perform these four checks in sequence:

**Check 1: Config Validation**
- Run `node src/config-validator.js` and capture exit code + output
- Report PASS if exit 0, FAIL with validation errors if exit 1
- This checks all config/*.json files against their schemas

**Check 2: Connector Reachability**
- Read `config/connectors.json` to identify configured connectors
- For each connector, check if its MCP server is listed as available by running `/mcp` status or `claude mcp list` (note: this may not be scriptable — if not, skip and report "manual check required")
- Alternative: check if the connector source files exist at their expected paths (src/connectors/gmail.js, src/connectors/github.js, src/connectors/calendar.js)
- Report each connector's status

**Check 3: Obsidian Local REST API**
- Run `curl -s -o /dev/null -w "%{http_code}" http://localhost:27123/` to check if the Obsidian REST API is responding
- Report REACHABLE if HTTP 200-399, UNREACHABLE if connection refused or 4xx/5xx
- Note: API may require auth header — a 401 still means the server is running (report as REACHABLE-AUTH-REQUIRED)

**Check 4: Scheduler State**
- Check if scheduling config exists by reading `config/scheduling.json`
- Report whether scheduling is configured and what the schedule parameters are
- Do NOT attempt to query Claude Desktop scheduled tasks (not scriptable)

**Output format** — the skill must instruct Claude to produce exactly this table:

```
Pipeline Health Report
======================

| Component        | Status      | Detail                          |
|------------------|-------------|---------------------------------|
| Config validity  | PASS/FAIL   | [validator output summary]      |
| Gmail connector  | OK/MISSING  | [file exists / file not found]  |
| GitHub connector | OK/MISSING  | [file exists / file not found]  |
| Calendar connector| OK/MISSING | [file exists / file not found]  |
| Vault REST API   | UP/DOWN     | [HTTP status or error]          |
| Scheduler config | OK/MISSING  | [schedule summary or not found] |

Overall: HEALTHY / DEGRADED (N issues) / UNHEALTHY (N critical)
```

Rules for the overall verdict:
- HEALTHY: all components OK/PASS/UP
- DEGRADED: 1-2 non-critical issues (e.g., one connector missing, scheduler not configured)
- UNHEALTHY: config validation fails OR vault REST API is down

The skill must complete all checks within 30 seconds. Include a note that individual curl/node commands have a 5-second timeout.

Keep the SKILL.md under 200 lines. Do not explain what Obsidian or MCP are — Claude already knows.
