  <action>
Edit `package.json` to add a new entry in the `scripts` block:

```json
"test:uat:ci": "jest --forceExit test/uat/"
```

The script does NOT set or clear the CI env var. The GitHub Actions workflow (Plan 17-01) sets `CI: ''` at the step level, which is the contract; the script is the bare jest invocation.

Place the new script immediately AFTER the existing `"test:uat"` line so related scripts cluster together. Final scripts block should read:

```json
"scripts": {
  "test": "jest --verbose",
  "lint": "eslint src/ test/",
  "test:uat": "CI= npx jest --forceExit test/uat/",
  "test:uat:ci": "jest --forceExit test/uat/",
  "license-check": "license-checker --production --onlyAllow 'MIT;ISC;Apache-2.0;BSD-2-Clause;BSD-3-Clause;CC0-1.0'"
}
```

Rationale for the difference between the two scripts:
- `test:uat` (local): uses `npx jest` with `CI=` prefix because Pete may have CI set in his shell env from other tooling; the prefix guarantees the skip guard doesn't trip locally.
- `test:uat:ci` (workflow): uses bare `jest` because the workflow step already controls CI via the step's `env:` block. Adding `CI=` to the npm script would be redundant and would mask the architectural contract (workflow owns the env, script owns the invocation).

Do NOT modify any other field in package.json (name, version, dependencies, jest config, etc.).
  </action>
