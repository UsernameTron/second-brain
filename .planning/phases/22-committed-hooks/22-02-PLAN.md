<objective>
Remove the `require('dotenv').config()` call from `src/pipeline-infra.js` (line 23) so that dotenv loading happens only at true entry points, not when a library module is imported. This eliminates the side effect that causes env var leakage in tests and violates the principle that library modules should not re-load environment at import time.

Purpose: `pipeline-infra.js` is imported by 13+ modules. Each `require('./pipeline-infra')` currently triggers `dotenv.config()`, which silently overwrites `process.env` values — a root cause of test flakiness and a violation of entry-point-only env loading discipline.

Output: Modified `pipeline-infra.js` without the dotenv side effect, updated tests that no longer need to mock dotenv, and a documented pattern for where dotenv loading belongs.
</objective>
